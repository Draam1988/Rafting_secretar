# Итоговый Протокол Спринта Design

## Цель

Добавить в раздел `Протоколы` первый реальный документ: `Итоговый протокол спринта`.

## Что должно быть в документе

- одна HTML-страница для просмотра и проверки;
- шапка соревнования:
  - организатор;
  - место проведения;
  - название соревнования;
  - даты соревнования;
  - название документа;
  - дисциплина;
- далее все категории соревнования подряд на одной странице;
- для каждой категории отдельный блок таблицы;
- в таблицу попадают только категории, где есть команды или результаты.

## Колонки таблицы

- `№ п/п`
- `№`
- `Команда`
- `Состав`
- `Субъект`
- `Время старта`
- `Время`
- `Штраф`
- `Примечание`
- `Место`
- `Очки`

## Источник данных

- `Команда`, `№`, `Субъект` берутся из команд;
- `Состав` берется не из общей заявки команды, а из стартового состава спринта;
- `Время старта`, `Время`, `Штраф`, `статус` берутся из спринта;
- `Место` вычисляется из ранжирования спринта;
- `Очки` вычисляются по таблице очков дисциплины `sprint`;
- `Примечание` строится из статуса:
  - `Финиш` -> пусто;
  - `Не старт`;
  - `Не финиш`;
  - `Дисквалификация`.

## UI блока `Протоколы`

- раздел `Протоколы` становится реестром документов;
- первым элементом списка будет `Итоговый протокол спринта`;
- остальные документы пока можно показать как `Скоро`, без ложной кликабельности.

## Ограничения первой версии

- пока только HTML-просмотр;
- без PDF и Excel экспорта;
- без отдельного печатного layout;
- без стартовых протоколов других дисциплин;
- без многоборья в этом шаге.
