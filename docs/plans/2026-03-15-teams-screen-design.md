# Teams Screen Design

**Goal**

Сделать экран `Команды` удобным для секретаря соревнований: он должен работать не с одной общей таблицей, а с независимыми блоками по выбранным категориям соревнования.

## Agreed Product Behavior

- Экран `Команды` строится только из категорий, выбранных в настройках соревнования.
- Каждая категория — отдельный независимый раскрываемый блок.
- Блоки идут вертикально, на всю ширину окна.
- В заголовке блока:
  - название категории
  - количество команд
  - кнопка `+ Добавить команду`
- Внутри раскрытого блока:
  - форма добавления команды
  - ниже список уже сохраненных команд этой категории

## Chosen UI Approach

Для первой версии выбран `вариант 1`:

- форма добавления команды находится прямо внутри блока категории;
- сохраненные команды показываются ниже карточками;
- все работает на одном экране, без ухода на отдельную страницу редактирования.

Это сознательно более прямой и наглядный путь. Он лучше подходит для первых живых тестов с секретарями, потому что интерфейс будет очевиден и без обучения.

## Team Form

### Team data

В карточке команды сверху идут основные поля:

- `Название команды`
- `Номер`
- `Клуб`
- `Региональная принадлежность`
- `Представитель команды (ФИО)`

### Crew data

Ниже идет блок `Состав команды`.

Для `R4`:
- `Участник 1`
- `Участник 2`
- `Участник 3`
- `Участник 4`
- `Запасной`

Для `R6`:
- `Участник 1`
- `Участник 2`
- `Участник 3`
- `Участник 4`
- `Участник 5`
- `Участник 6`
- `Запасной`

Для каждого участника:
- `ФИО`
- `Дата рождения`
- `Разряд`

## Saved Team Card

После сохранения команда показывается как компактная карточка:

- название
- номер
- клуб
- регион
- представитель
- количество заполненных участников

Карточка раскрывается и показывает полный состав команды.

## Actions On Saved Team

Сразу нужны оба действия:

- `Редактировать`
- `Удалить`

Удаление должно идти через обязательное подтверждение.

## Data Model Direction

Текущая модель `Team` и таблица `athletes` слишком простые. Для этого экрана нужен переход к более богатой структуре:

### Team

- id
- name
- start_number
- club
- region
- representative_full_name
- boat_class
- sex
- age_group

### Team member

- team_id
- member_order
- role (`main` / `reserve`)
- full_name
- birth_date
- rank

## Dashboard / Status

Для блока `Команды` на рабочем столе пока достаточно простого статуса:

- `red` не нужен;
- `yellow` если команд нет;
- `green` если есть хотя бы одна команда.

Короткий текст:
- `0 команд`
- `N команд`

## Constraints

- Минимализм без визуального шума.
- Не использовать перегруженную табличную разметку.
- Не смешивать категории между собой.
- Сохранять весь сценарий на одном экране.

## Testing

Нужны тесты на:

- корректный вывод категорий из настроек соревнования;
- правильный размер состава для `R4` и `R6`;
- сохранение команды с полями команды и участниками;
- вывод сохраненных карточек по категориям;
- редактирование команды;
- удаление команды с подтверждением.
