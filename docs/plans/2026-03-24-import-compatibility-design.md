# Import Compatibility Design

Дата: 2026-03-24

## Контекст

В проекте уже есть импорт и экспорт `.db` файла соревнования.
Сейчас приложение:
- блокирует импорт только когда схема базы новее поддерживаемой;
- не показывает отдельного предупреждения, если схема совместима, но файл создан другой версией приложения.

Это оставляет пользователя без понятного сигнала: файл можно открыть, но он был создан в другой версии RaftSecretary.

## Цель

Сделать поведение импорта прозрачным и безопасным:
- если схема базы новее — импорт блокируется;
- если схема совместима, но `app_version` отличается — импорт разрешается;
- после успешного импорта пользователь получает понятное предупреждение о другой версии приложения.

## Подходы

### Подход 1. Проверять только `schema_version`

Плюсы:
- просто;
- минимальный код.

Минусы:
- пользователь не понимает, что файл создан другой версией приложения.

### Подход 2. Проверять `schema_version` и информировать через `app_version`

Плюсы:
- безопасный технический критерий остается у схемы;
- пользователь получает ясный сигнал про различие версий;
- не ломаются старые базы без `app_version`.

Минусы:
- чуть больше логики на импорт и главную страницу.

### Подход 3. Жестко блокировать любую разницу `app_version`

Плюсы:
- очень строго.

Минусы:
- слишком много ложных блокировок;
- плохой UX для реально совместимых файлов.

## Решение

Выбран Подход 2.

`schema_version` остается главным критерием совместимости.
`app_version` используется как пользовательская информация:
- если схема несовместима, импорт запрещаем;
- если схема совместима, но версия приложения отличается, импортируем и показываем предупреждение;
- если метаданных версии нет, считаем файл старым форматом и импортируем без блокировки.

## Поведение UI

На стартовом экране после импорта:
- `import_error=incompatible` — красный баннер с блокировкой;
- `import_notice=version_mismatch` — нейтральный или мягкий предупреждающий баннер.

## Технические заметки

- Метаданные берем из `app_meta`.
- Логику проверки лучше держать в `raftsecretary/storage/db.py`.
- В `app.py` остается только интерпретация результата и показ баннера.
