# Sprint Screen Design

**Goal**

Сделать блок `Спринт` как рабочий экран секретаря, где он формирует стартовый протокол и вносит результаты заездов по категории.

## Core Product Rule

Экран `Спринт` не должен быть итоговым протоколом результатов.

Он нужен для оперативной работы:
- сформировать стартовый порядок;
- провести жеребьевку или ввести старт вручную;
- внести результаты заездов.

Итоговый протокол результатов должен жить отдельно в разделе `Протоколы`.

## Screen Structure

### 1. Header

- название соревнования
- дисциплина `Спринт`
- выбранная категория
- кнопка возврата на рабочий стол

### 2. Category switch

Если в соревновании несколько категорий, секретарь должен быстро переключать категорию.

### 3. Sprint actions

Сверху нужны действия:
- `Провести жеребьевку`
- `Пережеребить`

При этом секретарь может и вручную править стартовый порядок.

## Table Layout

Сразу показываем все команды категории в одной таблице.

Колонки:
- `№ старта`
- `Команда`
- `№ команды`
- `Состав`
- `Субъект РФ`
- `Время`
- `Штраф за неспортивное поведение`
- `Результат`
- `Статус`

## Data Entry Rules

### Start order

Стартовый порядок должен поддерживать два сценария:

1. Секретарь нажимает `Провести жеребьевку`
   - система случайно расставляет стартовые номера.

2. Секретарь вводит порядок вручную
   - система строит стартовый протокол по этим номерам.

После жеребьевки порядок не фиксируется жестко:
- можно вручную менять;
- можно делать `Пережеребить`.

## Result Entry

### Time

Вводится время прохождения дистанции.

### Penalty

Для `Спринта` отдельные штрафы за дистанцию не нужны.

Есть только:
- `штраф за неспортивное поведение`

По умолчанию:
- `0`

### Status

Статус вводится через выпадающий список в строке:
- `Финиш`
- `Не старт`
- `Не финиш`
- `Дисквалификация`

Внутри системы это можно маппить на служебные коды.

### Result

Итоговый результат считается автоматически:
- `время + штраф за неспортивное поведение`

## Table Behavior

- `Состав` должен быть мелким шрифтом, чтобы не раздувать таблицу.
- Результаты вводятся прямо в строках.
- Экран ориентирован на быстрый ввод, а не на красивый отчет.

## Protocols Split

### In `Спринт`

- стартовый протокол;
- оперативный ввод результатов.

### In `Протоколы -> Спринт`

- итоговый протокол результатов;
- сортировка по местам;
- позже:
  - очки
  - выполненный разряд
  - печатная форма.

## Testing

Нужны тесты на:
- жеребьевку;
- ручное изменение стартового порядка;
- сохранение времени и штрафа;
- автоматический расчет результата;
- статус через выпадающий список;
- отображение всех команд категории в одной таблице.
