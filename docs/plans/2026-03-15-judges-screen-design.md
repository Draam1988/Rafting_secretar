# Judges Screen Design

**Goal**

Сделать отдельный рабочий экран `Судьи` для секретаря соревнований, чтобы он мог заполнить обязательные роли и остальной судейский состав без ручной правки базы или технических полей.

**Context**

Сейчас на `/judges` есть только заглушка. На рабочем столе блок `Судьи` уже есть, но он не выполняет свою задачу и всегда показывает статус `Еще не реализовано`.

## Approaches

### Approach 1: One flat form

Один длинный экран со всеми судьями подряд.

Плюсы:
- быстро реализуется;
- не требует отдельной структуры страницы.

Минусы:
- обязательные роли теряются в общем списке;
- длинный экран хуже читается;
- сложнее быстро проверить, что критичные поля заполнены.

### Approach 2: Required roles + repeatable judges cards

Сверху отдельный блок `Обязательные роли`, ниже блок `Остальные судьи` с кнопкой `Добавить судью`.

Плюсы:
- обязательные роли всегда на виду;
- структура совпадает с реальной задачей секретаря;
- легко считать статус блока на рабочем столе.

Минусы:
- требует чуть больше логики в storage и UI.

### Approach 3: Tabbed interface

Разделить `Судьи` на вкладки `Обязательные` и `Список`.

Плюсы:
- экран короче.

Минусы:
- для MVP лишняя сложность;
- скрывает важные данные за переключением вкладок.

## Recommendation

Выбран `Approach 2`.

Это самый понятный рабочий вариант для MVP: обязательные роли видны сразу, остальные судьи добавляются по необходимости, экран остается минималистичным и не перегруженным.

## Screen Structure

### 1. Header

- заголовок `Судейский состав`
- короткое пояснение
- кнопка возврата на рабочий стол

### 2. Required roles

Отдельные карточки для:
- `Главный судья соревнований`
- `Главный секретарь соревнований`
- `Начальник дистанции`

В каждой карточке:
- `Фамилия`
- `Имя`
- `Отчество`
- `Судейская категория`

### 3. Other judges

- блок `Остальные судьи`
- кнопка `Добавить судью`
- повторяющиеся карточки судей

В каждой карточке:
- `Фамилия`
- `Имя`
- `Отчество`
- `Судейская категория`

## Data Model

Нужны две сущности:

1. `required_judges`
- ключ роли: `chief_judge`, `chief_secretary`, `course_chief`
- фамилия
- имя
- отчество
- категория

2. `judges`
- id
- фамилия
- имя
- отчество
- категория
- порядок отображения

## Category Options

Dropdown судейской категории:
- `Спортивный судья всероссийской категории`
- `Спортивный судья первой категории`
- `Спортивный судья второй категории`
- `Спортивный судья третьей категории`
- `Юный спортивный судья`

## Status Logic For Dashboard

Блок `Судьи`:
- `red` если не заполнена хотя бы одна обязательная роль
- `yellow` если обязательные роли заполнены, но нет остальных судей
- `green` если обязательные роли заполнены и есть хотя бы один судья в общем списке

Короткий текст:
- `Не заполнено`
- `Только обязательные роли`
- `Состав заполнен`

## Save Flow

- одна форма на экране `Судьи`
- `Сохранить судейский состав`
- после сохранения возврат на этот же экран
- значения формы должны подгружаться обратно из базы

## Error Handling

- если база соревнования не найдена, страница не должна падать
- пустые строки сохраняются безопасно
- отсутствие обычных судей допустимо, отсутствие обязательных ролей допустимо до завершения заполнения

## Testing

Нужны тесты на:
- создание и чтение обязательных ролей;
- создание и чтение списка обычных судей;
- сохранение через POST;
- повторную загрузку формы со значениями;
- статус блока `Судьи` на рабочем столе.
